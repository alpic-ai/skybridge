# Use skybridge/server

This guide shows you how to add `skybridge/server` to an existing MCP server to enable widget support with full TypeScript type inference.

## Prerequisites

You should already have:
- A working MCP server using the official [TypeScript MCP SDK](https://github.com/modelcontextprotocol/typescript-sdk)
- Node.js 24+

## Install Skybridge

```bash
pnpm add skybridge
```

## Update your server

Replace your `McpServer` import with Skybridge's enhanced version:

```typescript
// Before
import { McpServer } from "@modelcontextprotocol/sdk/server/mcp.js";

// After
import { McpServer } from "skybridge/server";
```

Keep your existing dependency to @modelcontextprotocol/sdk in your project, skybridge does not replaces it, but depends on it.

Your existing tools, resources, and prompts will continue to work as before. With `skybridge/server`, `registerTool` gains an optional `view` field so you can bind a tool to a React widget in a single call.

## Project structure

Skybridge uses a flat `src/` layout with widgets in `src/views/`:

```
your-project/
├── src/
│   ├── server.ts             # MCP server entry
│   └── views/
│       ├── my-widget.tsx     # Widget component
│       └── other-widget.tsx
├── vite.config.ts            # Vite + Skybridge plugin
├── tsconfig.json
└── package.json
```

### Widget naming convention

A widget's file name must match `view.component` in `registerTool` (e.g., `component: "my-widget"` → `src/views/my-widget.tsx`). See [registerTool](/api-reference/register-tool).

## Set up Vite

Install the web dependencies and create `vite.config.ts` at the project root:

```bash
pnpm add react react-dom
pnpm add -D vite typescript @types/react @types/react-dom
```

```typescript
// vite.config.ts
import { defineConfig } from "vite";
import { skybridge } from "skybridge/web";

export default defineConfig({
  plugins: [skybridge()],
});
```

The plugin auto-mounts every file in `src/views/` — no `mountWidget` call needed.

Make sure `tsconfig.json` picks up the generated types:

```json
{
  "include": ["src", ".skybridge/**/*.d.ts"]
}
```

## Create your first widget

Create `src/views/my-widget.tsx` and `export default` a React component:

```tsx
export default function MyWidget() {
  return (
    <div>
      <h2>Hello from Skybridge!</h2>
      <p>Your widget is working!</p>
    </div>
  );
}
```

## Register the widget

In your server code, register the widget with `registerTool` + `view`:

```typescript
import { McpServer } from "skybridge/server";
import { z } from "zod";

const server = new McpServer({ name: "my-app", version: "1.0" }, {})
  .registerTool(
    {
      name: "my-widget",
      description: "My first widget",
      inputSchema: { message: z.string() },
      view: { component: "my-widget" }, // must match src/views/my-widget.tsx
    },
    async ({ message }) => {
      return { content: message || "Hello World" };
    },
  );
```

## Configure your dev server

Update your server startup to serve the MCP endpoint. If you're using Express:

```typescript
import { McpServer } from "skybridge/server";
import { z } from "zod";
import express from "express";

const app = express();
const server = new McpServer({ name: "my-app", version: "1.0" }, {})
  .registerTool(
    {
      name: "my-widget",
      inputSchema: { message: z.string() },
      view: { component: "my-widget" },
    },
    async ({ message }) => {
      return { content: message };
    },
  );

// Serve the MCP endpoint
app.use("/mcp", server.handler());

// Start the server
app.listen(3000, () => {
  console.log("Server running on http://localhost:3000");
});
```

## Type Safety

For full type inference and autocomplete, use method chaining and `generateHelpers`. See [Type Safety](/concepts/type-safety) for the complete setup.

Quick summary:
1. Use method chaining when registering tools
2. Export `type AppType = typeof server`
3. Create a `helpers.ts` file with `generateHelpers<AppType>()`
4. Import typed hooks from your helper file
