---
title: mcpAuthMetadataRouter
description: "Advertise your authorization server for client discovery"
---

When a client reaches your server without a token, it needs to know where the user signs in. `mcpAuthMetadataRouter` advertises that, so clients can discover your authorization server on their own.

## Example

The server publishes where to authorize, so a client hitting a 401 can find the authorization server on its own.

```ts server.ts highlight={4-13}
import { McpServer, mcpAuthMetadataRouter } from "skybridge/server";

const server = new McpServer({ name: "shop", version: "1.0" }).use(
  mcpAuthMetadataRouter({
    oauthMetadata: {
      issuer: "https://auth.example.com",
      authorization_endpoint: "https://auth.example.com/authorize",
      token_endpoint: "https://auth.example.com/token",
      response_types_supported: ["code"],
    },
    resourceServerUrl: new URL("https://api.example.com/mcp"),
    scopesSupported: ["shop.read"],
  }),
);
```

## Signature

```ts
mcpAuthMetadataRouter(options: AuthMetadataOptions): Router;
```

## Parameters

### `options`

```ts
type AuthMetadataOptions = {
  oauthMetadata: OAuthMetadata; // your authorization server's metadata (RFC 8414)
  resourceServerUrl: URL; // the URL of this MCP server
  scopesSupported?: string[]; // scopes this server recognizes
  serviceDocumentationUrl?: URL; // link to human-readable docs for this server
  resourceName?: string; // display name for this resource in the metadata
};
```

- `oauthMetadata` is your authorization server's [RFC 8414](https://datatracker.ietf.org/doc/html/rfc8414) metadata: at minimum the `issuer`, the `authorization_endpoint` and `token_endpoint`, and `response_types_supported`, as the example shows.
- `resourceServerUrl` is published in the protected-resource metadata so clients map this server to that authorization server.

<Info>
Check your OAuth provider's docs for the metadata values it expects.
</Info>

## Returns

An Express `Router` to pass to [`server.use`](/api-reference/mcp-server#use). It serves your OAuth 2.0 [Protected Resource Metadata](https://datatracker.ietf.org/doc/html/rfc9728) at `/.well-known/oauth-protected-resource`.
