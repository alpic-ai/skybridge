`FileRef` is a reference to a file, an id and a URL to fetch its bytes. Use it in a [tool's `inputSchema` or `outputSchema`](/api-reference/register-tool#inputschema-outputschema) to pass files in and out.

<Info>
File params are a **ChatGPT** feature. Other hosts don't pass files through [tool](/build/tools) calls, so a `FileRef` field is populated only on ChatGPT.
</Info>

## Example

`summarize-document` takes a file in, fetches its bytes from `download_url`, and returns a summary file back.

```ts server.ts highlight={6-7}
import { FileRef, McpServer } from "skybridge/server";

const server = new McpServer({ name: "docs", version: "1.0" }).registerTool(
  {
    name: "summarize-document",
    inputSchema: { document: FileRef },
    outputSchema: { summary: FileRef },
    _meta: { "openai/fileParams": ["document"] },
  },
  async ({ document }) => {
    const bytes = await fetch(document.download_url).then((r) => r.blob());
    const summary = await summarize(bytes); // returns a FileRef
    return { structuredContent: { summary } };
  },
);
```

## Shape

```ts
type FileRef = {
  file_id: string;
  download_url: string;
  mime_type?: string;
  file_name?: string;
};
```

| Field | Purpose |
| --- | --- |
| `file_id` | The host's identifier for the file. |
| `download_url` | A URL to fetch the file's bytes. |
| `mime_type` | The file's MIME type, when known. |
| `file_name` | The original file name, when known. |

`download_url` is required in both positions: on input the host fills it, on output your handler produces a URL the host can fetch.
